Product Management API Documentation

This api has been developed by C2S

This documentation outlines the functionalities of the Product Management API, which interacts with a MongoDB database. All endpoints use JSON for data exchange.

Endpoints

GET /api/products

Retrieves a list of all products stored in the database.

Response (200 OK)

[
  {
    "id": "12345",
    "name": "T-Shirt",
    "description": "Comfortable cotton t-shirt",
    "price": 19.99
  },
  {
    "id": "67890",
    "name": "Laptop",
    "description": "High-performance laptop for work and gaming",
    "price": 899.99
  }
]

POST /api/products

Creates a new product and inserts it into the database.

Request Body

{
  "name": "Coffee Mug",
  "description": "Durable ceramic mug with cool design",
  "price": 9.99
}

Response (201 Created)

{
  "id": "abcdef",
  "name": "Coffee Mug",
  "description": "Durable ceramic mug with cool design",
  "price": 9.99
}

Response (400 Bad Request)

Returned if the request body is invalid.

PUT /api/products/:id

Modifies an existing product based on its unique identifier (id).

Request Body

{
  "name": "Laptop (Updated)",
  "description": "High-performance laptop with upgraded graphics card",
  "price": 999.99
}

Response (200 OK)

{
  "id": "67890",
  "name": "Laptop (Updated)",
  "description": "High-performance laptop with upgraded graphics card",
  "price": 999.99
}

Response (400 Bad Request)

Returned if the request body is invalid.

Response (404 Not Found)

Returned if the product with the specified id is not found.

DELETE /api/products/:id

Deletes a product from the database based on its unique identifier (id).

Response (204 No Content)

Returned upon successful deletion.

Response (404 Not Found)

Returned if the product with the specified id is not found.
